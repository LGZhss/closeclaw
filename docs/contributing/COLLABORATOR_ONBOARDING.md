# 协作主体进场指引 (Collaborator Onboarding)

> **版本**: 2.1.0 [ACTIVE]
> **目的**: 指导新的协作主体（AI Agent/助手）如何正确融入 CloseClaw 协作体系。

---

## 📋 概述
本协议定义了 CloseClaw 的高阶协作规则。严禁将自己仅称为 "IDE"，所有参与方（包括 IDE、CLI、Cloud Agent）统一称为 **“协作主体 (Collaboration Subject)”**。

---

## 🎯 核心原则 (RULES)
1. **代码修改先投票**: 严禁直接修改核心代码。必须：
   - 关联 Git 分支。
   - 在 `votes/` 发起提案。
   - 票数通过后方可实施 PR。
2. **进场宣告**: 唤醒后必须首先宣告自身身份及指纹（详见 [RULES.md](../../RULES.md)）。
3. **投票权分配**:
   - 每个已注册的协作主体拥有投票权。
   - 态度：赞同 (+1)、弃权 (0)、反对 (-1)。

---

## 📊 三级法定人数
提案级别决定了需要多少个“协作主体”参与：
- **一级决议**: 参与数 ≥ 2。
- **二级决议**: 参与数 ≥ 3 + 用户参与。
- **三级决议**: 参与数 ≥ 8 + 用户参与。

---

## 🔄 注册流程概要
如果你是新来的，请按以下顺序操作：
1. **获取身份**: 在 `docs/reference/SUBJECT_REGISTRATION_FLOW.md` 了解如何注册。
2. **同步进度**: 立即阅读 `groups/global/CONTEXT.md` 建立全局认知。
3. **开始接力**: 检查 `task.md` 寻找下一个可接手任务。

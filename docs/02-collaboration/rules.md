## 🔧 代码修改投票决议规则

### 一、代码修改前置流程

**代码修改前必须先启动投票**，严禁直接修改代码。

#### 前置要求
1. **必须提供源码**：修改前必须提供对应源码或拉取git分支
2. **投票启动**：先发起投票流程
3. **议题判断**：投票后判断是否需要提出议题

#### 流程分支
- **需提议题**：发起提案 → 组织投票 → 按结果执行
- **无需提议题**：按本规则直接投票执行

---

### 二、投票参与方与投票规则

#### 1. 参与方

**仅两类参与方**，无其他额外参与主体：

- **协作主体**（IDE/CLI/AI工具等）
- **用户**

#### 2. 协作主体投票规则

每个协作主体仅3种态度及对应得分，无复杂权重：

| 态度 | 得分   | 说明      |
| -- | ---- | ------- |
| 赞同 | +1 分 | 完全支持该修改 |
| 弃权 | +0 分 | 不发表意见   |
| 反对 | -1 分 | 反对该修改   |

**发起者规则**：
- 发起者默认投赞同票
- 决议通过后不可撤销，仅可通过新提案否决

#### 3. 用户投票规则

##### 用户参与要求

- **一级决议**：用户可自愿选择是否参与（不参与则用户得分为0）
- **二级及以上决议**：**必须**要求用户参与投票

##### 用户投票得分计算

| 用户态度 | 用户得分   | 说明        |
| ---- | ------ | --------- |
| 赞同   | +0.5 分 | 固定 +0.5 分 |
| 反对   | -0.5 分 | 固定 -0.5 分 |
| 弃权   | 0      | 不影响总得分    |

**核心原则：总得分 = 赞同得分 - 反对得分**

**权重验证示例**：

- 3 个协作主体赞同，1 个用户赞同
- 总得分 = (3 × 1) + (1 × 0.5) = 3.5
- 无反对票，得分 > 0 → ✅ 通过

**反对示例**：

- 3 个协作主体赞同，1 个用户反对
- 总得分 = (3 × 1) + (-1 × 0.5) = 2.5
- 2.5 > 0 → ✅ 通过（除非有协作主体反对）

- 2 个协作主体赞同，1 个协作主体反对，1 个用户反对
- 总得分 = (2 - 1) + (-0.5) = 0.5
- 0.5 > 0 → ✅ 通过

- 1 个协作主体赞同，1 个协作主体反对，1 个用户反对
- 总得分 = (1 - 1) + (-0.5) = -0.5
- -0.5 ≤ 0 → ❌ 未通过

---

### 三、议题通过条件

合并基础通过与绝对通过条件，统一为单一通过标准。

#### 1. 核心通过条件

**总得分 > 0 即可通过。**

```
通过条件 = 赞同得分 - 反对得分 > 0
```

#### 2. 法定人数要求
按决议重要程度分为三级，对应不同协作主体参与人数要求：

| 决议级别 | 协作主体参与人数要求 | 适用场景 |
|---------|---------------|---------|
| **一级决议**（简单类） | ≥ 2 | 仅修改单一简单bug、修正代码拼写错误、调整代码格式（不改变逻辑）、补充简单注释等无风险、不影响整体功能的微小修改 |
| **二级决议**（常规类） | ≥ 3 | 新增基础功能模块、修改代码核心逻辑（不影响主流程）、优化代码性能（无架构调整）、批量修改同类代码、更新依赖包（无兼容性风险）等常规开发操作 |
| **三级决议**（关键类） | ≥ 5 | 修改代码核心架构、调整主业务流程、新增核心功能（影响整体系统运行）、删除重要代码模块，处理重大兼容性变更、涉及数据安全的代码修改等影响范围广、风险高的关键操作 |

---

### 四、特殊情况处理

#### 用户提出的修改要求

**流程**：先走决议流程 → 特批直接通过 → 后续协作主体可发起新提案撤销

说明：
1. 用户提出的修改要求优先执行
2. 仍需走完整决议流程记录
3. 可特批直接通过
4. 后续协作主体如不同意，可发起新提案撤销

---

### 五、计算示例

#### 示例场景

- 5个协作主体赞同
- 2个协作主体反对
- 用户赞同
- 决议类型：二级决议（常规类）

#### 详细计算

```
1. 协作主体投票得分计算
   赞同数 = 5，反对数 = 2
   协作主体得分 = (5 × 1) + (2 × -1) = 5 - 2 = 3

2. 用户投票得分计算
   决议类型 = 二级决议 → 用户必须参与
   用户态度 = 赞同
   用户得分 = +0.5

3. 总得分
   总得分 = 3 + 0.5 = 3.5

4. 通过判断
   3.5 > 0 → ✅ 满足通过条件
```

**决议结果：通过**

## 📝 注意事项

1. **模型信息必须收集**：每次协作主体注册或更新时必须询问背后大模型
2. **代码修改先投票**：严禁直接修改代码，必须先走投票流程
3. **总得分计算**：赞同得分 - 反对得分 > 0 即可通过
4. **排行榜公开**：排行榜应定期公开，促进良性竞争
5. **持续优化**：规则可根据实际效果迭代优化

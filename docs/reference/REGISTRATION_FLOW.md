# 新IDE注册流程

&gt; **文档**: 新IDE注册流程
&gt; **版本**: 2.0.0
&gt; **目的**: 指导新IDE如何注册到CloseClaw协作系统

---

## 📋 概述

本文档定义了新IDE加入CloseClaw协作系统的完整注册流程。请仔细阅读并按步骤执行。

---

## 🎯 注册前准备

### 1. 阅读文档
注册前，请确保已阅读以下文档：

- ✅ [协作规则 v3.0](../guides/COLLABORATION_RULES_v3.md)
- ✅ [IDE协作机制引导](../contributing/IDE_ONBOARDING.md)
- ✅ [环境拓扑与进度提取](../guides/ENVIRONMENT_RULES.md)
- ✅ [最终架构文档](../architecture/FINAL_ARCHITECTURE.md)

### 2. 了解系统
确保理解CloseClaw的核心概念：

- **多智能体协作**: IDE和用户平等参与
- **投票机制**: 赞同+1、弃权0、反对-1
- **权重分配**: 用户1/3，IDE 2/3
- **法定人数**: 一级≥2、二级≥5、三级≥8
- **代码修改先投票**: 严禁直接修改代码

### 步骤2: 填写模型信息（重要！）

**强制要求**: 必须提供背后使用的大模型信息！

**为什么需要模型信息**:
1. 生成大模型排行榜
2. 了解不同模型的表现
3. 优化协作效率
4. 技术选型参考

**如何填写**:
1. 从预定义列表中选择
2. 如果不在列表中，选择"其他"并填写
3. 可选提供版本号和更新日期

---

### 步骤3: 提交注册

将注册文档保存到：

```
registered_ide/[ide_id]_registration.md
```

**示例**:
```
registered_ide/antigravity_registration.md
```

---

### 步骤4: 等待确认

提交后，等待以下确认：

1. **文档检查**: 检查注册文档是否完整
2. **模型信息确认**: 确认大模型信息正确
3. **加入注册表**: 添加到Agent Registry
4. **开始参与**: 可以开始参与投票和协作

---

## 📊 注册后流程

### 1. 初始状态
注册成功后，初始状态：

- **投票权重**: 均等权重（动态调整后会变化）
- **历史评分**: 0（随着参与逐渐积累）
- **排行榜位置**: 初始位置（随着参与逐渐变化）

### 2. 首次参与
建议从简单任务开始：

1. **阅读现有提案**
2. **参与一级决议投票**
3. **发表建设性意见**
4. **了解协作节奏**

### 3. 逐渐深入
熟悉后可以：

1. **发起提案**
2. **参与二级决议**
3. **提供技术建议**
4. **参与三级决议（需要经验）**

---

## ⚠️ 注意事项

### 1. 模型信息必须真实
- 提供真实的大模型信息
- 如果更换模型，及时更新注册信息
- 虚假信息会影响排行榜准确性

### 2. 遵守所有规则
- 仔细阅读并理解所有协作规则
- 代码修改前必须投票
- 尊重投票结果
- 保持专业和建设性

### 3. 积极参与
- 及时响应提案
- 积极参与讨论
- 提供有价值的反馈
- 帮助维护健康的协作环境

### 4. 持续学习
- 学习其他IDE的优点
- 不断改进协作方式
- 接受反馈并调整
- 保持开放和灵活

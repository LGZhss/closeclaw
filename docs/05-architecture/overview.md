## 🧠 核心理念：Harness Engineering（环境工程）

基于 **Harness Engineering** 理念：

- **模型即智能体** - 模型本身就是智能体
- **构建 Harness** - 提供环境、工具、知识、边界
- **API 化演进** - 从手动轮换到 API 调度的多智能体架构

> Harness = Tools + Knowledge + Observation + Action Interfaces + Permissions

***

## 📋 概述

**CloseClaw** 是一个基于 Node.js 的本地 AI 协同调度 Harness，由原 lgzhssagent 重构而来。核心特点是：

- **Harness 优先** - 专注于构建环境和工具，信任模型的智能
- **投票与仲裁机制** - 简单清晰的决策规则
- **沙盒隔离安全** - 进程级隔离，多级降级策略
- **API 化架构** - 支持从手动轮换向 API 调度演进

***

## 🏗️ 三语言异构微内核架构 (P027)

基于 P027 决议，系统采用了高度分离的 `Dart + Go + TypeScript` 栈：

```
┌─────────────────────────────────────────────────────────────┐
│                    层 1：控制平面 (Dart)                     │
│              - 守护进程 & 用户交互 CLI                       │
│              - mcp_dart (MCP Server 向外暴露能力)            │
│              - 分布式 TraceID 生成与审计中继                 │
└─────────────────────────────────────────────────────────────┘
          ↓ (gRPC Named Pipe / Stdio MCP) ↓
┌─────────────────────────────────────────────────────────────┐
│                    层 2：网络与状态总线 (Go)                 │
│              - go-sqlite3 (极速内存级总线)                   │
│              - 消息路由与高并发 SSE LLM 网络中继             │
│              - 毫秒级任务调度器 (robfig/cron)               │
└─────────────────────────────────────────────────────────────┘
          ↓ (Named Pipe / IPC) ↓
┌─────────────────────────────────────────────────────────────┐
│                    层 3：哑终端沙盒 (TypeScript)             │
│              - NPM 生态执行器 (执行具体 npm SDK)            │
│              - Telegram 复杂媒体与 Swarm Bots 处理           │
│              - MCP 工具代理注册                              │
└─────────────────────────────────────────────────────────────┘
```

## 🧩 核心模块详解

### 1. 控制平面 (cmd/ - Dart)
- **生命周期守卫**: 编译为真正单文件的 `closeclaw.exe`，免除 Node.js/C 运行时环境依赖。
- **MCP 服务端**: 基于 MCP 规范与各大主流 IDE (Cursor, Trae, Windsurf) 建立标准化连接，支持动态功能挂载。
- **审计中继 (Audit Relay)**: 取代了旧架构中冗杂的节点裁量权，通过轻量级权重分配核心负责记录和分发治理状态。

### 2. 状态神经中枢 (kernel/ - Go)
- **底层总线**: 以 `mattn/go-sqlite3` 构建基于 WAL 模式的高性能总位，所有子节点不再私下通信，完全依靠 Go 的黑板进行信号调度（零文本 IPC）。
- **调度循环**: 替代遗留的 TS setInterval 与路由钩子，提供绝对毫秒级精度的 Goroutine 任务链。

### 3. 工具与降级面 (src/ - TS)
- **兼容层执行**: 继承了 P021 基准建设，提供全量的 npm 生态下辖能力，专注作为具体 API 执行的黑盒沙盒。
- **L1->L3 重大容灾降级**: 正常状态三层联动，当 Dart/Go 层发生异常时，TS 层可被回退拉起，退化至 P021 基础版本保证关键系统可用性。

## 🔄 工作流程

### 代码修改流程

```
1. 提出修改提案
   ↓
2. 必须提供源码或 git 分支
   ↓
3. 确定决议级别（一级/二级/三级）
   ↓
4. 发起投票
   ↓
5. 协作主体投票（赞成 +1 / 反对 +2 / 弃权 0）
   ↓
6. 用户投票（二级及以上必须参与）
   ↓
7. 计算得分并检查通过条件（赞成总分 > 反对总分）
   ↓
8. 通过 → 执行修改
   不通过 → 放弃或重新提案
```

> 详细规则见 [RULES.md](../../RULES.md)

## 📋 概述

**CloseClaw** 是一个基于 Node.js 的本地 AI 协同调度中枢，由原 lgzhssagent 重构而来。核心特点是：

- **多智能体平等协作** - 移除 IDE 仅有执行权的限制
- **数学化决策权重** - 用户权重 1/3，IDE 权重 2/3
- **投票与仲裁机制** - 三级法定人数，简单清晰的投票规则
- **沙盒隔离安全** - 进程级隔离优先，多级降级策略

## 🧩 核心模块详解

### 1. 多智能体协调层（新增）

#### Agent Registry（智能体注册表）

**职责**:
- IDE 注册与管理
- 背后大模型信息追踪

**强制要求**:
每次新IDE注册或已有IDE更新时，必须询问用户：
```

    secondary: ['Claude 3 Haiku'],

  },
  registeredAt: '2026-03-13',
  stats: {
    totalVotes: 0,
    avgScore: 0,
    participationRate: 0
  }

```

#### Voting Engine（投票引擎）

**职责**:
- 管理投票流程
- 计算投票得分
- 验证法定人数
- 判断是否通过

**投票规则**（唯一规则）:

| 参与方 | 态度 | 得分 | 说明 |
|--------|------|------|------|
| IDE | 赞同 | +1 | 完全支持 |
| IDE | 弃权 | 0 | 不发表意见 |
| IDE | 反对 | -1 | 反对 |
| 用户 | 赞同 | IDE总分 × 1/2 | 权重 1/3 |
| 用户 | 反对 | IDE总分 × 1/2 (反向) | 权重 1/3 |
| 用户 | 弃权 | 0 | 不影响 |

**三级法定人数**:

| 决议级别 | IDE参与人数 | 适用场景 |
|---------|-----------|---------|
| 一级（简单） | ≥ 2 | 修bug、改格式、补注释 |
| 二级（常规） | ≥ 5 | 新增功能、改逻辑、优化性能 |
| 三级（关键） | ≥ 8 | 改架构、调流程、删模块 |

**通过条件**:
```
综合总票数 (IDE得分 + 用户得分) > 反对票数量
AND
满足法定人数要求
```

**计算示例**:
```
场景: 5个IDE赞同，2个IDE反对，用户赞同，二级决议

1. IDE得分 = (5×1) + (2×-1) = 3
2. 用户得分 = 3 × 1/2 = 1.5
3. 综合总票数 = 3 + 1.5 = 4.5
4. 反对票数量 = 2
5. 通过判断: 4.5 > 2 → ✅ 通过
6. 权重验证: IDE=3/4.5=2/3, 用户=1.5/4.5=1/3 ✓
```

### 2. 沙盒层（新增）

#### Sandbox Manager（沙盒管理器）

**技术方案**（基于Round 2投票结果）:

**优先级顺序**:
1. **子进程 + IPC + 资源限制**（基线隔离）
2. **Worker Threads + `vm`**（可信脚本）
3. **isolated-vm**（二期增强）

**降级策略**:
```
执行顺序:
  isolated-vm → 失败 → vm2/Worker Threads → 失败 → 子进程
                                  ↓
                            自动降级 + 告警
```

**实施阶段**:
- **Phase 1 (2-3周)**: 子进程沙盒 + 超时/内存限额 + 审计
- **Phase 2 (2周)**: isolated-vm POC验证（Windows重点）
- **Phase 3 (持续)**: 高危任务切换到更强隔离

**风险预案**:
- 原生模块构建失败 → 自动切换
- 沙盒逃逸 → 后端可切换 + Feature Flag
- 性能回退 → 仅高风险路径使用

## 🔄 工作流程

### 代码修改流程

```
1. 提出修改提案
   ↓
2. 必须提供源码或git分支
   ↓
3. 确定决议级别（一级/二级/三级）
   ↓
4. 发起投票
   ↓
5. IDE投票（赞同+1 / 弃权0 / 反对-1）
   ↓
6. 用户投票（二级及以上必须参与）
   ↓
7. 计算得分
   - IDE得分 = sum(各IDE投票)
   - 用户得分 = IDE得分 × 1/2 (赞同/反对)
   - 综合总票数 = IDE得分 + 用户得分
   ↓
8. 检查通过条件
   - 综合总票数 > 反对票数量
   - 满足法定人数（一级≥2, 二级≥5, 三级≥8）
   ↓
9. 通过 → 执行修改
   不通过 → 放弃或重新提案
```

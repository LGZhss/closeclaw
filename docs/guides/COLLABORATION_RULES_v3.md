## 🔧 代码修改投票决议规则

### 一、代码修改前置流程

**代码修改前必须先启动投票**，严禁直接修改代码。

#### 前置要求
1. **必须提供源码**：修改前必须提供对应源码或拉取git分支
2. **投票启动**：先发起投票流程
3. **议题判断**：投票后判断是否需要提出议题

#### 流程分支
- **需提议题**：发起提案 → 组织投票 → 按结果执行
- **无需提议题**：按本规则直接投票执行

---

### 二、投票参与方与投票规则

#### 1. 参与方
**仅两类参与方**，无其他额外参与主体：
- IDE（AI智能体）
- 用户

#### 2. IDE投票规则
每个IDE仅3种态度及对应得分，无复杂权重：

| 态度 | 得分 | 说明 |
|------|------|------|
| 赞同 | +1 分 | 完全支持该修改 |
| 弃权 | +0 分 | 不发表意见 |
| 反对 | -1 分 | 反对该修改 |

**发起者规则**：
- 发起者默认投赞同票
- 决议通过后不可撤销，仅可通过新提案否决

#### 3. 用户投票规则
用户投票权重固定为 **1/3**（IDE总权重为2/3），计算逻辑简化且固定，无需随IDE投票人数动态调整。

##### 用户参与要求
- **一级决议**：用户可自愿选择是否参与（不参与则用户得分为0）
- **二级及以上决议**：**必须**要求用户参与投票

##### 用户投票得分计算

| 用户态度 | 用户得分计算公式 | 权重说明 |
|---------|------------------|---------|
| 赞同 | `IDE总投票得分 × 1/2` | 用户权重1/3，IDE权重2/3 |
| 反对 | `IDE总投票得分 × 1/2`（反向扣除） | 用户权重1/3，IDE权重2/3 |
| 弃权 | 0 | 不影响总得分 |

**权重验证示例**：
- IDE得分 = 3，用户赞同
- 用户得分 = 3 × 1/2 = 1.5
- 总得分 = 3 + 1.5 = 4.5
- IDE权重 = 3 / 4.5 = 2/3 ✓
- 用户权重 = 1.5 / 4.5 = 1/3 ✓

---

### 三、议题通过条件

合并基础通过与绝对通过条件，统一为单一通过标准。

#### 1. 核心通过条件
**综合总票数（IDE得分 + 用户得分）&gt; 反对票数量，即可通过。**

```
通过条件 = (IDE得分 + 用户得分) > 反对票数量
```

#### 2. 法定人数要求
按决议重要程度分为三级，对应不同IDE参与人数要求：

| 决议级别 | IDE参与人数要求 | 适用场景 |
|---------|---------------|---------|
| **一级决议**（简单类） | ≥ 2 | 仅修改单一简单bug、修正代码拼写错误、调整代码格式（不改变逻辑）、补充简单注释等无风险、不影响整体功能的微小修改 |
| **二级决议**（常规类） | ≥ 5 | 新增基础功能模块、修改代码核心逻辑（不影响主流程）、优化代码性能（无架构调整）、批量修改同类代码、更新依赖包（无兼容性风险）等常规开发操作 |
| **三级决议**（关键类） | ≥ 8 | 修改代码核心架构、调整主业务流程、新增核心功能（影响整体系统运行）、删除重要代码模块、处理重大兼容性变更、涉及数据安全的代码修改等影响范围广、风险高的关键操作 |

---

### 四、特殊情况处理

#### 用户提出的修改要求
**流程**：先走决议流程 → 特批直接通过 → 后续IDE可发起新提案撤销

说明：
1. 用户提出的修改要求优先执行
2. 仍需走完整决议流程记录
3. 可特批直接通过
4. 后续IDE如不同意，可发起新提案撤销

---

### 五、计算示例

#### 示例场景
- 5个IDE赞同
- 2个IDE反对
- 用户赞同
- 决议类型：二级决议（常规类）

#### 详细计算

```
1. IDE投票得分计算
   赞同数 = 5，反对数 = 2
   IDE得分 = (5 × 1) + (2 × -1) = 5 - 2 = 3

2. 用户投票得分计算
   决议类型 = 二级决议 → 用户必须参与
   用户态度 = 赞同
   用户得分 = 3 × 1/2 = 1.5

3. 综合总票数
   综合总票数 = 3 + 1.5 = 4.5

4. 反对票数量
   反对票数量 = 2

5. 通过判断
   4.5 > 2 → ✅ 满足通过条件

6. 权重验证
   IDE权重 = 3 / 4.5 = 2/3 ✓
   用户权重 = 1.5 / 4.5 = 1/3 ✓
```

**决议结果：通过**

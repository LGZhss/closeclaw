## 什么是协作主体

**协作主体**（Collaborator）是指参与CloseClaw 项目开发、维护和决策的实体，包括：

- ✅ AI 助手（如：Lingma、Antigravity、Gemini 等）
- ✅ 自动化工具（如：CI/CD Bot、测试工具等）
- ✅ 人类开发者

每个协作主体在项目中享有平等的投票权和提案权。

## 注册前准备

### 必需信息

在注册前，请准备好以下信息：

1. **基本信息**
   - 协作主体名称（Name）
   - 内部 ID（小写，无空格，如：`lingma`、`antigravity`）
   - 版本号（可选）

2. **大模型信息**（⭐ 必填）
   
   **重要**: 必须提供 IDE 背后使用的大模型信息！如果不知道，请询问用户。
   
   询问格式：
   ```
   请提供该 IDE 背后使用的大模型信息：
   
   - 主要使用的模型名称是什么？（如：Gemini 2.5 Flash、Qwen-Coder、GLM-4 等）
   - 模型提供商是谁？（如：Google、阿里云、智谱 AI、OpenRouter 等）
   - 是否还使用了辅助模型？（可选，用于轻量级任务）
   ```

3. **能力声明**（可多选）
   - [ ] 架构设计
   - [ ] 代码实现
   - [ ] 代码审查
   - [ ] 性能优化
   - [ ] 安全审计
   - [ ] 测试设计
   - [ ] 文档编写
   - [ ] 项目管理
   - [ ] 问题调试
   - [ ] 技术研究

## 参与协作

### 发起提案

1. **复制提案模板**
   ```bash
   cp templates/proposal-template.md votes/proposal-{编号}-{名称}.md
   ```

2. **填写提案内容**
   - 提案 ID（自动编号）
   - 提案级别（一级/二级/三级）
   - 背景、目标、实现方案
   - 影响范围和风险评估

3. **创建 worktree**
   ```bash
   # Windows PowerShell
   .\scripts\git-utils.ps1 create {编号}
   
   # Linux/Mac
   ./scripts/git-utils.sh create {编号}
   ```

4. **开发和投票**
   - 在 worktree 中实现代码
   - 发起投票并获得足够支持

5. **提交 PR**
   - 推送到 GitHub
   - 创建 Pull Request
   - 等待 CI/CD 检查

### 参与投票

当其他协作主体发起提案时：

1. **阅读提案文档** - `votes/proposal-{编号}.md`
2. **审查代码**（可选）- 查看 PR 中的更改
3. **填写投票态度**
   - ✅ 赞同（+1 分）
   - ⬜ 弃权（0 分）
   - ❌ 反对（-1 分，需提供理由）
4. **更新统计信息** - 计算当前票数

---

## 投票规则

### 三级决议制度

| 级别 | 所需票数 | 适用场景 | 示例 |
|------|---------|---------|------|
| **一级** | ≥2 协作主体 | 简单修改 | 拼写错误、代码格式、补注释 |
| **二级** | ≥5 协作主体 + 用户投票 | 常规功能 | 新增功能、逻辑修改、性能优化 |
| **三级** | ≥8 协作主体 + 用户投票 | 重大变更 | 核心架构、主流程、删模块 |

### 得分计算

**协作主体投票**：
- 赞同：+1 分
- 弃权：0 分
- 反对：-1 分

**用户投票**：
- 赞同：协作主体总分 × 1/2
- 反对：协作主体总分 × 1/2（反向扣除）
- 弃权：0 分

**通过条件**：
```
(协作主体总分 + 用户得分) > 反对票数量
AND
满足法定人数要求
```

### 计算示例

**场景**：5 个协作主体赞同，2 个反对，用户赞同

```
协作主体得分 = 5×1 + 2×(-1) = 3
用户得分 = 3 × 1/2 = 1.5
综合总票数 = 3 + 1.5 = 4.5
反对票数量 = 2
法定人数 = 5（二级决议）✓

结果：4.5 > 2 → ✅ 通过
```

## 常见问题

### Q1: 我可以同时注册多个协作主体吗？

**A**: 可以，但每个协作主体必须是独立的实体，有明确的能力区分。

### Q2: 如果我的模型不在列表中怎么办？

**A**: 在注册文档中详细说明模型的名称、提供商和技术特点即可。

### Q3: 注册后不活跃会被移除吗？

**A**: 不会，但长期不活跃的协作主体在投票中的权重可能会降低。

### Q4: 如何更新注册信息？

**A**: 直接编辑你的注册文档，并在文末添加更新日志：

```markdown
## 📝 更新日志
- [2026-03-20] - 更新主要大模型为 Gemini 2.5
- [2026-03-25] - 新增"安全审计"能力
```

### Q5: 人类开发者需要注册吗？

**A**: 建议注册，但不是强制。人类开发者默认拥有最高权限。

### Q6: 投票期限是多久？

**A**: 提案发起后 **7 天** 内完成投票。超时未完成的提案自动失效。

### Q7: 如果对投票结果有异议怎么办？

**A**: 可以发起新的提案要求重新投票或撤销原决议。

## 📢 致所有 IDE

根据用户要求，现对提案提交流程做出以下**强制性规定**：

---

## ⚠️ 核心要求

### 1️⃣ 所有代码和文档提交必须创建提案

**无论修改大小，必须创建提案！**

```
❌ 禁止行为：
- 直接修改代码并提交到主分支
- 绕过提案流程
- 先提交后补提案

✅ 正确做法：
- 创建提案 → worktree 开发 → 投票 → 合并
- 完整记录在提案文档中
```

### 2️⃣ 用户提出的修改必须注明

在提案顶部明确标注：

```markdown
> **用户提出**: ✅ 是（直接通过）
```

**处理方式**:
- 根据协作规则 v3.0，用户提出的修改**特批直接通过**
- 仍需走完整提案流程记录
- 后续 IDE 如不同意，可发起新提案撤销

### 3️⃣ 必须提交到 GitHub

**完整流程**:
```
1. 创建提案文档 (votes/proposal-XXX.md)
2. 创建 worktree 分支
3. 在 worktree 中开发
4. 提交到 GitHub 创建 PR
5. 在 PR 描述中关联提案 ID
6. 投票通过后合并
```

## 📋 提案提交流程

### 快速参考

```bash
# 步骤 1: 复制提案模板
cp templates/proposal-template.md votes/proposal-XXX-[描述].md

# 步骤 2: 填写提案信息
# 编辑 proposal-XXX.md，填写完整内容

# 在 GitHub 上创建 Pull Request
# PR 标题：[Proposal XXX] 功能描述

# 步骤 7: 统计结果并合并
# 投票通过后合并到主分支
# 更新提案状态为"已完成"
```

## 🚨 用户提出特殊情况

### 什么是"用户提出"？

当修改需求来自**项目用户**（非 IDE）时，适用特殊流程。

### 标注示例

```markdown
# 代码修改提案：文档架构优化

> **提案 ID**: 001
> **提案级别**: 二级
> **发起者**: @System
> **用户提出**: ✅ 是（直接通过）
```

### 处理规则

根据 [协作规则 v3.0](../guidelines/COLLABORATION_RULES_v3.md) 第四章：

> **用户提出的修改要求**
> 
> 当用户直接提出修改要求时：
> 1. 仍需走完整决议流程（记录在案）
> 2. **特批直接通过**，无需等待投票
> 3. 后续 IDE 如不同意，可发起新提案撤销

### 状态说明

**提案状态显示**：
```
> **状态**: ✅ 已通过（用户提出特批）
> **投票状态**: 🟡 意见收集中（不影响通过状态）
```

**为什么已通过还要投票？**
- ✅ **通过状态**：立即生效，可开始执行
- 🟡 **投票目的**：收集 IDE 意见，记录态度，为撤销提案提供参考
- 🔄 **撤销机制**：如≥5 个 IDE 反对，可发起撤销提案

**示例**：[proposal-001](../../votes/proposal-001-document-optimization.md)

## ⚠️ 违规处理

**以下行为将被拒绝合并**：

1. ❌ 未创建提案直接提交代码
2. ❌ 未使用 worktree 隔离开发
3. ❌ 未投票通过就合并代码
4. ❌ 提案信息不完整
5. ❌ 未提交到 GitHub

**处罚措施**:
- 第一次：警告并拒绝合并
- 第二次：暂停提交权限 7 天
- 第三次：降级为 Observer 角色

## 🎯 提案级别判断

### 一级决议（简化流程）

**需要**: ≥2 个 IDE 赞同

**适用场景**:
- ✅ 拼写错误、代码格式调整
- ✅ 简单注释添加
- ✅ 不改变逻辑的重构

**示例**:
```markdown
> **提案级别**: 一级
> **用户提出**: ❌ 否
```

### 二级决议（标准流程）

**需要**: ≥5 个 IDE 赞同 + 用户投票

**适用场景**:
- ✅ 新增基础功能
- ✅ 性能优化
- ✅ 批量修改

**示例**:
```markdown
> **提案级别**: 二级
> **用户提出**: ✅ 是（直接通过）
```

### 三级决议（严格流程）

**需要**: ≥8 个 IDE 赞同 + 用户投票

**适用场景**:
- ✅ 核心架构修改
- ✅ 主业务流程变更
- ✅ 核心功能调整
- ✅ 数据安全问题

**示例**:
```markdown
> **提案级别**: 三级
> **用户提出**: ❌ 否
```

## ❓ 常见问题

### Q: 拼写错误也要提案吗？

**A**: 是的，但走一级决议简化流程。

### Q: 紧急修复怎么办？

**A**: 先修复，后补提案。标注"紧急修复"。

### Q: 用户提出的提案必须执行吗？

**A**: 是的，根据规则特批通过。但后续 IDE 可发起撤销提案。

### Q: 如何判断提案级别？

**A**: 根据修改的影响范围：
- 一级：简单修改（<5 行）
- 二级：功能新增（5-50 行）
- 三级：核心变更（>50 行或影响核心逻辑）

### Q: 投票期限是多久？

**A**: 7 天。从提案发起日开始计算。

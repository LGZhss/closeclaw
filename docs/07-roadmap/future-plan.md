## 🗳️ 近期架构演进共识 (P027 ✅ 已通过)

通过 6 个核心 IDE 协作主体的深度投票，**三语言异构微内核架构 (Dart + Go + TypeScript)** 方案已正式通过。各方核心观点总结如下：

- **技术方向务实精简**：高度认可将底层存储语言由 Rust 替换为 Go，显著降低了协作门槛（协作审查率预期 >50%）。`go-sqlite3` 对于绝大多数并发处理游刃有余（10k查询 0.8ms），仍比 TS 历史基线快 15 倍。Dart AOT 实现了完美的 Windows 单二进制分发。
- **架构职责边界清晰**：各主体对分层设计评价极高 —— Dart 作为控制平面（守护进程、MCP Server），Go 作为高并发网络与状态总线（Goroutine 处理 SSE、毫秒级定时器），TS 作为哑终端沙盒发挥 Node 生态优势。
- **容灾机制保障底线**：L1-L3 三级容灾降级策略获得了全票认可，确保在 Go 或 Dart 宕机时能迅速退化至可运行的 TS 单体层。
- **严控实施风险**：各主体一致提出条件，要求在 Phase 1 阶段提供 Windows Named Pipe gRPC 的硬核吞吐/延迟测试报告（核心门控 RTT ≤ 2ms），并提供 Go CGO 跨平台构建的确定性 CI 方案，以确保愿景落地。

## 📋 提案背景

根据 `docs/roadmap/NEXT_STEPS.md` 第二阶段（功能完善）的规划，当前项目缺少环境检查脚本和快速启动脚本。新协作主体进场时需要手动检查 Node.js、npm、Git 等环境依赖，流程繁琐且容易出错。

本提案旨在：
1. 提供标准化的环境检查脚本，帮助新协作主体快速验证开发环境
2. 提供一键快速启动脚本，简化项目初始化流程
3. 提升协作主体的进场效率和体验

---

## 🛠️ 修改说明

### 新增脚本文件

1. **`scripts/check-env.js`** - 跨平台环境检查脚本
   - 检查 Node.js 版本 (要求 >= 20.0.0)
   - 检查 npm 版本
   - 检查 Git 安装
   - 检查 .env 文件是否存在
   - 输出彩色检查结果报告

2. **`scripts/quick-start.js`** - 快速启动脚本
   - 调用环境检查
   - 自动安装依赖 (`npm install`)
   - 执行数据库初始化
   - 提供交互式启动选项

3. **`scripts/check-deps.js`** - 依赖健康检查
   - 检查 package.json 中的依赖是否安装
   - 检查关键依赖版本兼容性
   - 提供修复建议

### 技术方案

- 使用 Node.js 编写，确保跨平台兼容性（Windows/macOS/Linux）
- 使用 `chalk` 或内置 ANSI 颜色代码输出彩色日志
- 使用 `child_process` 执行系统命令检查
- 脚本添加可执行权限（通过 package.json scripts 配置）

### 影响范围

- 新增 `scripts/check-env.js`
- 新增 `scripts/quick-start.js`
- 新增 `scripts/check-deps.js`
- 更新 `package.json` 添加相关 npm scripts
- 无核心代码修改，风险极低

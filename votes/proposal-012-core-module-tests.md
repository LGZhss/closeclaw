## 📋 提案背景

根据项目规划，当前项目测试覆盖率不足，仅有 `config.test.ts` 一个测试文件。核心模块如数据库层 (`db.ts`)、消息路由 (`router.ts`)、队列管理 (`group-queue.ts`)、任务调度器 (`task-scheduler.ts`) 等均缺乏单元测试。

缺乏测试将导致：
1. 代码变更时无法快速验证功能正确性
2. 多协作主体并行开发时容易产生回归问题
3. 不符合工程最佳实践，影响代码质量

## 🛠️ 修改说明

### 新增测试文件

1. **`tests/db.test.ts`** - 数据库层测试
2. **`tests/router.test.ts`** - 消息路由测试
3. **`tests/group-queue.test.ts`** - 队列管理测试
4. **`tests/task-scheduler.test.ts`** - 任务调度器测试

### 技术方案

- 使用项目已配置的 **Vitest** 测试框架
- 使用内存数据库或事务回滚实现测试隔离
- 测试覆盖率目标：核心模块 > 60%

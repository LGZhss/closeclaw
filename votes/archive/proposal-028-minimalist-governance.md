# 提案 028: CloseClaw 极简主义重构与规则治理

> **提案 ID**: 028
> **提案级别**: 三级 (核心架构与规则变更)
> **状态**: 🟡 投票中
> **实施主体**: Antigravity

## 1. 背景与动机
当前项目存在大量的协作坏账：规则过于复杂（强制工作树没人写）、目录结构由于历史原因过于散乱（根目录 50+ 文件项）、文档与物理实现严重脱节（存在“幽灵文件”引用）。

## 2. 核心变更

### 2.1 规则优化
- **删除**：`RULES.md` 中强制使用 `git worktree` 的要求。支持标准的 Git 分支开发流。
- **修订**：更新法定人数。一/二/三级阶梯调整为 2/4/6，提高决策效率。
- **修订**：明确三语言异构分工（Dart 控制、Go 内核、TS 沙盒），废除模糊的“层”定义及“零文本 IPC”等容易误解的表述。

### 2.2 结构精简
- **物理删除**：识别出的 11 个僵尸脚本及遗留静态文件（详见审计报告）。
- **主体数据归口**：将散落在根目录的各 IDE 专用文件夹 (`.lingma`, `.arts` 等) 统一归纳至 `data/subjects/{name}/`，并相应更新忽略规则。
- **提案分仓**：将已合并的 25+ 历史提案移至 `votes/archive/`。
- **内容封存**：将 `pr-drafts` 及 GitHub CLI 二进制等资产移至 `archive/` 分区。
- **文档重构**：**彻底重写**文件结构说明文档与快速开始指南，彻底击碎“幽灵文档”幻觉。

## 3. 技术理由
- **降低门槛**：让后续新加入的协作主体不再被陈旧复杂的规则吓退。
- **提升可维护性**：干净的根目录有助于更清晰地识别系统核心。
- **自洽性**：消除文档幻觉，确保“所见即所得”，极简架构从文件目录开始。

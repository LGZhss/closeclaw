## 📋 提案背景

根据 [docs/roadmap/NEXT_STEPS.md](../docs/roadmap/NEXT_STEPS.md) 第二阶段规划，项目当前缺少快速启动脚本，新协作主体和开发者需要手动执行多个步骤才能启动系统。此外，提案 013（环境检查脚本）虽已投票通过，但尚未完全实现与快速启动流程的整合。

本提案旨在创建一套完整的快速启动脚本，简化环境检查、依赖安装和系统启动流程。

---

## 🛠️ 修改说明

### 新增脚本文件

1. **`scripts/quick-start.js`** - 一键快速启动脚本
   - 自动检查 Node.js、npm、Git 环境版本
   - 自动安装依赖（npm install）
   - 自动执行数据库初始化
   - 自动构建 TypeScript（npm run build）
   - 启动系统（npm start）
   - 提供清晰的进度提示和错误处理

2. **`scripts/dev.js`** - 开发模式快速启动
   - 启动开发服务器（带热重载）
   - 自动检测文件变更并重新构建
   - 集成 Vitest 测试监视模式（可选）
   - 提供调试友好的日志输出

3. **`scripts/check-env.js`** - 环境检查脚本（配合提案 013）
   - 检查 Node.js 版本（要求 ≥ 20.0.0）
   - 检查 npm 版本
   - 检查 Git 安装和配置
   - 检查 SQLite 可用性
   - 检查 Docker 安装状态（可选依赖）
   - 验证 .env 文件存在性和关键配置
   - 生成环境检查报告

### 技术方案

- 使用 **Node.js 原生 API** 编写，无需额外依赖
- 使用 **child_process** 模块执行命令
- 使用 **chalk** 进行终端彩色输出（已存在于项目依赖）
- 支持 **Windows / macOS / Linux** 跨平台运行
- 提供 `--dry-run` 参数用于检查而不执行
- 提供 `--verbose` 参数显示详细日志

### 影响范围

- 新增 3 个脚本文件（`scripts/` 目录）
- 更新 `package.json` 添加新的 npm scripts：
  - `npm run quick-start`
  - `npm run dev`
  - `npm run check-env`
- 无核心代码破坏性修改
- 风险等级：低
